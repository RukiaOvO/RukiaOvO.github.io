`Kg4se` 是我围绕 **软件工程知识组织与利用** 搭建的知识图谱平台。这个项目面向课程与研究场景，我尝试把文档解析、知识抽取、图谱构建、问答增强与质量评估串成一条完整链路，而不是只停留在单点功能演示上。

从仓库结构和 README 可以看出，我把它设计成了一个相对完整的全栈系统：

- 前端使用 **Vue 3 + TypeScript + Vite**，配合 **Naive UI、ECharts、Cytoscape.js** 完成图谱展示、统计可视化与交互页面
- 后端使用 **FastAPI + Python 3.11**，负责文档处理、知识抽取、GraphRAG 流水线和问答服务
- 基础设施引入 **Neo4j、Redis、RQ、Docker Compose**，让图数据存储、异步处理和本地部署都具备了工程化基础

这个项目里我比较看重的一点，是我没有把“知识图谱”简单理解成图数据库展示，而是明确设计了多阶段的 **GraphRAG pipeline**，并把文档管理、图谱构建、知识卡片、问答和评估模块拆开处理。这让系统既有研究导向，也保留了较强的工程可扩展性。

如果用一句话概括，`Kg4se` 更像是我对 **软件工程课程知识进行结构化、图谱化、可视化，并进一步服务于问答与分析** 的一次平台化尝试，而不是单纯的图数据库 demo。
